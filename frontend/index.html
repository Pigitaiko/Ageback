Step-by-Step Demo

Walk through the full agent → x402 payment → cashback flow. Each step shows what happens under the hood.

1

Request API (Get 402)

Send a request to the x402-gated endpoint without payment. The server returns HTTP 402 with payment requirements.

2

Sign x402 Payment

The agent's x402 client signs a USDC transferWithAuthorization (EIP-3009) to pay the provider.

How it works in production: The @x402/evm client library intercepts the 402 response, reads the payment requirements, signs a gasless USDC authorization, and retries the request with the payment header attached. No wallet popup is needed because agents hold their own private keys.
3

Pay & Get Response + Cashback

The server verifies payment, proxies to Claude API, and allocates on-chain cashback from the provider's rebate pool.

Run It Yourself

To run the full end-to-end flow with a real USDC payment and on-chain cashback allocation, use the test agent script:

cd x402-server && node test-agent.js

What test-agent.js does:
1. Loads agent wallet with USDC on Taiko Hoodi
2. Creates an x402-enabled HTTP client with wrapAxios()
3. POSTs to /v1/messages — gets HTTP 402
4. x402 client auto-signs a transferWithAuthorization and retries
5. Server verifies USDC payment via the facilitator
6. Server proxies request to Claude API and returns the response
7. Server calls RebatePoolManager.allocateRebate() on-chain
8. Agent receives cashback proportional to the provider's rebate rate

Quick start

        

          
          
          
        


        

          
Add cashback to any x402 Express server in 5 lines:

          
npm install @ageback/middleware ethers express \
  @x402/express @x402/core @x402/evm

          
import { attachAgeback } from "@ageback/middleware";

const ageback = attachAgeback(app, {
  rpc: process.env.TAIKO_RPC,
  rebatePoolManager: process.env.REBATE_POOL_MANAGER,
  providerPrivateKey: process.env.PROVIDER_PRIVATE_KEY,
  service: { name: "My API", category: "data" },
});

app.use(paymentMiddleware({
  "POST /my-route": {
    accepts: [ageback.buildAccepts({ price: "$0.01", payTo: PAY_TO })],
    description: "Paid route with Ageback cashback",
    mimeType: "application/json",
  },
}, resourceServer, undefined, undefined, true));

app.post("/my-route", (req, res) => {
  ageback.allocateForRequest(req, res, { paymentAmountUsd: "0.01" });
  res.json({ ok: true });
});

        


        

          
Agents already paying with x402 get cashback automatically. For rich UX:

          
npm install @ageback/client

          
import { createAgebackClient } from "@ageback/client";

const ageback = createAgebackClient({ paidFetch }); // your @x402 fetch
const result = await ageback.fetch("https://ageback.onrender.com/v1/messages", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: "claude-haiku-4-5-20251001",
    max_tokens: 50,
    messages: [{ role: "user", content: "hi" }],
  }),
});
console.log(result.cashback);
// { enabled: true, bps: 300, rebatePercent: "3.00%",
//   agent: "0x...", contract: "0x...", txHash: "0x..." }

        


        

          
Three HTTP endpoints — no RPC or ABI required:

          
# Service-level metadata (name, chain, contracts, rebate %)
curl https://ageback.onrender.com/.well-known/ageback.json

# Every Ageback-enabled provider, one HTTP call
curl https://ageback.onrender.com/providers

# Recent RebateAllocated events as JSON (provider, agent, amount, txHash, explorerUrl)
curl 'https://ageback.onrender.com/feed/cashback?limit=200'

          
x402 402 responses also self-advertise the cashback layer via the
          extensions["ageback.v1"] field — read it directly from the 402.

ageback.v1 extension schema

        

          Service providers include this object inside each accepts entry
          so x402 clients and explorers can discover cashback metadata directly from the 402.
        

        

          

            
              
FieldTypeRequiredNotes

            
            
              
rebatePoolManageraddressyesContract holding the rebate pool

              
provideraddressyesOn-chain provider entry the server settles to

              
rebateBpsnumberrecommendedActive rate in basis points (300 = 3%)

              
manifeststringrecommendedPath or URL to /.well-known/ageback.json

              
loyaltyTierManageraddressoptionalIf the provider records tier progression

              
referralGraphaddressoptionalIf the provider records referral volume

              
modelstringoptionalFor multi-tier APIs (e.g. Claude model id)
