Step-by-Step Demo

Walk through the full agent → x402 payment → cashback flow. Each step shows what happens under the hood.

1

Request API (Get 402)

Send a request to the x402-gated endpoint without payment. The server returns HTTP 402 with payment requirements.

2

Sign x402 Payment

The agent's x402 client signs a USDC transferWithAuthorization (EIP-3009) to pay the provider.

How it works in production: The @x402/evm client library intercepts the 402 response, reads the payment requirements, signs a gasless USDC authorization, and retries the request with the payment header attached. No wallet popup is needed because agents hold their own private keys.
3

Pay & Get Response + Cashback

The server verifies payment, proxies to Claude API, and allocates on-chain cashback from the provider's rebate pool.

Run It Yourself

To run the full end-to-end flow with a real USDC payment and on-chain cashback allocation, use the test agent script:

cd x402-server && node test-agent.js

What test-agent.js does:
1. Loads agent wallet with USDC on Taiko Hoodi
2. Creates an x402-enabled HTTP client with wrapAxios()
3. POSTs to /v1/messages — gets HTTP 402
4. x402 client auto-signs a transferWithAuthorization and retries
5. Server verifies USDC payment via the facilitator
6. Server proxies request to Claude API and returns the response
7. Server calls RebatePoolManager.allocateRebate() on-chain
8. Agent receives cashback proportional to the provider's rebate rate
